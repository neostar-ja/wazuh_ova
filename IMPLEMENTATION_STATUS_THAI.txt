📞 SUPPORT & TROUBLESHOOTING (การสนับสนุนและแก้ไขปัญหา)
═══════════════════════════════════════════════════════════════════════════════

ปัญหา: ไม่พบ alerts
───────────────────
✓ ตรวจสอบว่า rules ถูกติดตั้งหรือไม่
  ssh root@10.251.151.13 "grep -c '<rule' /var/ossec/etc/rules/1003-network-anomaly-rules.xml"
✓ ตรวจสอบ Wazuh Manager status
  ssh root@10.251.151.13 "sudo /var/ossec/bin/wazuh-control status"
✓ Restart Wazuh Manager
  ssh root@10.251.151.13 "sudo /var/ossec/bin/wazuh-control restart"

ปัญหา: SSH connection refused
──────────────────────────────
✓ ตรวจสอบ Manager ทำงานหรือไม่
  ping 10.251.151.13
✓ ตรวจสอบ SSH port 22
  ssh -v root@10.251.151.13
✓ ตรวจสอบ credentials
  ssh-keygen -t rsa (ถ้าต้องการ key-based auth)

ปัญหา: Permission denied errors
──────────────────────────────
✓ ตรวจสอบ file permissions
  ssh root@10.251.151.13 "ls -l /var/ossec/etc/rules/1003-network-anomaly-rules.xml"
✓ ตั้งค่า permissions ใหม่
  ssh root@10.251.151.13 "sudo chown root:ossec /var/ossec/etc/rules/1003-network-anomaly-rules.xml && sudo chmod 640 /var/ossec/etc/rules/1003-network-anomaly-rules.xml"
