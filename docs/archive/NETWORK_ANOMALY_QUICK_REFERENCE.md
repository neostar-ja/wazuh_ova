# Quick Reference - Network Anomaly Detection

## What This Does

Detects and alerts on:
- ✅ Internal IP → External IP with suspicious ports (SSH 22, RDP 3389, etc.)
- ✅ External IP → Internal IP with suspicious ports (attack attempts)
- ✅ Both DENY and PERMIT policies
- ✅ Brute force patterns (5+ attempts in 5 minutes)
- ✅ Sends Telegram alerts in real-time

## Installation (One Command)

```bash
sudo /opt/code/wazuh_ova/setup_network_anomaly_detection.sh
```

## After Installation

```bash
# 1. Restart Wazuh
sudo /var/ossec/bin/wazuh-control restart

# 2. Test the rules (automatic)
python3 /opt/code/wazuh_ova/generate_network_anomaly_tests.py

# 3. Check results
python3 /opt/code/wazuh_ova/test_network_anomaly_detection.py

# 4. View in Wazuh Dashboard
# Query: rule.groups: "network_anomaly"
```

## What Gets Detected

| Traffic Type | Port | Policy | Severity |
|---|---|---|---|
| Internal→External SSH | 22 | DENY | Medium |
| Internal→External SSH | 22 | PERMIT | High |
| External→Internal SSH | 22 | DENY | High |
| **External→Internal SSH** | **22** | **PERMIT** | **🔴 CRITICAL** |
| External→Internal RDP | 3389 | DENY | High |
| External→Internal RDP | 3389 | PERMIT | High |
| Inbound Telnet | 23 | Any | Medium |
| Outbound VNC | 5900 | Any | Medium |
| SSH Brute Force | 22 | Any | 🔴 CRITICAL |
| RDP Brute Force | 3389 | Any | 🔴 CRITICAL |

## Troubleshooting

**No alerts?**
- Check Wazuh status: `sudo /var/ossec/bin/wazuh-control status`
- Restart: `sudo /var/ossec/bin/wazuh-control restart`
- Check logs: `tail /var/ossec/logs/ossec.log`

**Telegram not working?**
- Verify token: `python3 -c "import requests; print('OK')"`
- Check credentials in script
- Test manually: `python3 /var/ossec/active-response/bin/telegram_network_alert.py`

**Too many false positives?**
- Review `COMPANY_INTERNAL_RANGES` in detector script
- Adjust rule severity levels
- Add whitelist for known management IPs
